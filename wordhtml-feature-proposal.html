เอกสารนี้สรุปจากการสำรวจโค้ดทั้งโปรเจคด้วยเอเจนต์ 3 ตัว เพื่อ ไม่เสนอของที่มีอยู่แล้ว (เช่น TOC, นับคำ, Find&Replace, mail-merge, ตัดย่อหน้าข้ามหน้า, ตัวช่วยภาษาไทย ฯลฯ ทำเสร็จหมดแล้ว) แต่เสนอ 3 กลุ่ม: (A) เก็บงานที่สร้างค้างไว้ให้จบ · (B) ทำให้เหมือน Microsoft Word มากขึ้น · (C) ระบบงานเอกสารราชการไทยโดยเฉพาะ
ลำดับ: P1 ทำก่อน · คุ้มสุด P2 ทำตามมา P3 ทำทีหลัง · ใหญ่/เสี่ยง ความพยายาม: ★ เล็ก · ★★ กลาง · ★★★ ใหญ่

A เก็บงานที่สร้างค้างไว้ให้จบ (Finish)

โค้ดมีโครงไว้แล้วบางส่วน — เก็บให้ครบจะได้ฟีเจอร์เร็วและเสี่ยงน้อย

A1 แก้ไขหัว/ท้ายกระดาษแบบพิมพ์ได้จริง (Header/Footer rich-text editing) P1
ตอนนี้กรอกหัว/ท้ายได้ผ่านกล่องข้อความใน dialog เท่านั้น แต่ node สำหรับแก้แบบ WYSIWYG มีอยู่แล้ว (pageHeader.ts / pageFooter.ts) เพียงแต่ยังไม่ได้ต่อเข้าโครงสร้างหน้าและไม่มีปุ่มเปิดโหมดแก้ → ทำให้แก้หัว/ท้ายได้เหมือนพิมพ์ในเนื้อหา (จัดฟอนต์ ใส่โลโก้ เลขหน้า) ตรงในกระดาษ
คุณค่า: สูง — งานราชการต้องมีหัวกระดาษ/ตราครุฑ/เลขหน้าเกือบทุกฉบับ พยายาม: ★★ ✓ ทำงานแบบ static ได้ แตะ: pageNode.ts, RibbonTabLayout, HeaderFooterDialog
A2 หน้าแรก / หน้าคู่-คี่ ต่างกันจริง (Different first / odd-even layout) P2
มีช่องตั้งค่า "หน้าแรกต่าง" และ "คู่-คี่ต่าง" ใน dialog แล้ว (HeaderFooterDialog.tsx) และมีตัว resolve (headerFooterResolve.ts) — แต่ปัจจุบันเปลี่ยนแค่ "เนื้อหาหัว/ท้าย" ยังไม่เปลี่ยน layout จริง → ทำให้หน้าแรกเว้นหัวกระดาษ/ระยะขอบต่างได้ (เช่น หน้าแรกมีตราครุฑเต็ม หน้าถัดไปย่อ)
คุณค่า: กลาง-สูง สำหรับเอกสารทางการ พยายาม: ★★ ✓ static แตะ: pageNode.ts, pagination layout
A3 ตัดตาราง/รูปข้ามหน้า (Table/Image splitting) P3
ปัจจุบันตาราง/รูปที่ยาวเกินหน้า จะถูกย้ายทั้งก้อนไปหน้าถัดไป (atomic) ทำให้เหลือที่ว่างด้านล่าง → ขยาย splitter.ts ให้ตัดกลางตารางได้ (พร้อมซ้ำหัวตารางบนหน้าใหม่) เหมือน Word
คุณค่า: กลาง พยายาม: ★★★ ⚠ เสี่ยง — กระทบ engine จัดหน้า ควรมีเทสต์แน่น แตะ: pagination/splitter.ts, engine.ts

B ทำให้เหมือน Microsoft Word (Parity)

ฟีเจอร์ที่ผู้ใช้ Word คุ้นเคยแต่แอปยังไม่มี — ส่วนใหญ่ทำได้ฝั่ง client ล้วน

B1 เชิงอรรถ / อ้างอิงท้ายเรื่อง (Footnotes / Endnotes) P1
ใส่ตัวเลขยกกำลังในเนื้อหา แล้วแสดงข้อความเชิงอรรถท้ายหน้า/ท้ายเอกสาร พร้อมเลขอัตโนมัติที่ปรับเมื่อเพิ่ม/ลบ → จำเป็นมากกับรายงานและเอกสารวิชาการ/ราชการ
คุณค่า: สูงพยายาม: ★★ ✓ static เพิ่ม: Tiptap node ใหม่ + export ทุกฟอร์แมต
B2 คำบรรยายรูป/ตาราง + เลขอัตโนมัติ (Captions) P1
เพิ่ม "รูปที่ 1", "ตารางที่ 2" ใต้รูป/ตาราง พร้อมเลขรันอัตโนมัติและอ้างอิงข้าม ("ดังรูปที่ 1") → ต่อยอดจาก imageWithAlign.ts ที่มีอยู่
คุณค่า: สูง (รายงานราชการ)พยายาม: ★★ ✓ staticแตะ: imageWithAlign.ts, node ใหม่
B3 ชุดสไตล์ย่อหน้า / แกลเลอรีสไตล์ (Style Gallery) P1
นิยามสไตล์ใช้ซ้ำ (ปกติ / หัวข้อ 1-3 / ชื่อเรื่อง / คำพูดอ้างอิง) แล้วคลิกเดียวเปลี่ยนทั้งย่อหน้าให้สม่ำเสมอทั้งเอกสาร → ลดงานจัดฟอร์แมตซ้ำ ๆ และทำให้เอกสารดูเป็นระเบียบแบบมืออาชีพ
คุณค่า: สูงพยายาม: ★★ ✓ static ต่อยอด: paragraphFormat.ts + กลุ่มใหม่ใน RibbonTabHome
B4 ลายน้ำ "ร่าง / สำเนา / ลับ" (Watermark) P1
ใส่ลายน้ำข้อความหรือรูปทแยงกลางหน้า ("ร่าง", "สำเนาถูกต้อง", "ลับ") แสดงในทุกหน้าและติดไปกับ PDF/พิมพ์ → คุณค่าสูงมากต่อราชการ ทำได้ง่ายด้วย CSS overlay
คุณค่า: สูงพยายาม: ★ (เล็ก คุ้มสุดในกลุ่ม) ✓ staticแตะ: page setup + print/PDF css
B5 คอมเมนต์ / หมายเหตุ (Comments / Annotations) P2
ไฮไลต์ข้อความแล้วแนบคอมเมนต์ แสดงเป็นแถบด้านข้าง — เก็บลงไฟล์โปรเจค .wordhtml.json ได้เลย (ไม่ต้องมี server) → ดีสำหรับตรวจทาน/ส่งกลับแก้
คุณค่า: สูง (งานเป็นทีม)พยายาม: ★★★ ✓ static (เก็บใน project json)ต่อยอด: project.ts
B6 จัดหลายคอลัมน์ (Multi-column layout) P2
แบ่งเนื้อหาเป็น 2-3 คอลัมน์แบบหนังสือพิมพ์/แผ่นพับ ผ่าน CSS columns ระดับ section
คุณค่า: กลางพยายาม: ★★ ✓ static
B7 บุ๊กมาร์ก / อ้างอิงข้าม (Bookmarks / Cross-references) P2
ปักหมุดตำแหน่งในเอกสาร แล้วสร้างลิงก์ "ดูข้อ 3" / "ดูหน้า 5" ที่อัปเดตเลขเองเมื่อเนื้อหาขยับ → ต่อยอดจากระบบ id หัวข้อที่มีใน headingWithId.ts
คุณค่า: กลางพยายาม: ★★ ✓ staticต่อยอด: headingWithId.ts, toc.ts
B8 ติดตามการแก้ไข (Track Changes) P3
แสดงข้อความที่เพิ่ม/ลบเป็นสีและขีดฆ่า พร้อมปุ่มยอมรับ/ปฏิเสธทีละจุด — เป็นฟีเจอร์ใหญ่และซับซ้อนที่สุด
คุณค่า: สูง แต่เฉพาะกลุ่มพยายาม: ★★★ ⚠ ใหญ่/ซับซ้อน — แนะนำทำทีหลัง
B9 การอ้างอิง/บรรณานุกรม · กล่องข้อความลอย (Citations · Text boxes) P3
จัดการรายการอ้างอิงและสร้างบรรณานุกรมท้ายเล่ม / กล่องข้อความลอยที่จัดวางอิสระ — ลำดับความสำคัญต่ำสำหรับงานทั่วไป
คุณค่า: เฉพาะกลุ่มวิชาการพยายาม: ★★–★★★

C ระบบงานเอกสารราชการไทย (Thai-gov)

จุดแข็งเฉพาะตัวของแอปนี้ — ต่อยอดจากตัวช่วยภาษาไทยและ template gallery ที่มีอยู่แล้ว

C1 ตัวช่วยร่างหนังสือราชการ (Thai official-letter wizard) P1
ฟอร์มกรอกทีละช่อง (เลขที่หนังสือ · ส่วนราชการ · วันที่ · เรื่อง · เรียน · อ้างถึง · สิ่งที่ส่งมาด้วย · เนื้อความ) แล้วสร้างเอกสารตามระเบียบงานสารบรรณให้ถูกฟอร์แมตทันที (หนังสือภายนอก/บันทึกข้อความ/ประกาศ/คำสั่ง) → คุณค่าสูงสุดในเอกสารนี้ ต่อยอดจาก templateGallery.ts + /procurement
คุณค่า: สูงสุดพยายาม: ★★ ✓ staticต่อยอด: templateGallery.ts, mergeFields
C2 บล็อกลงนาม + เลขหนังสืออัตโนมัติ (Signature block + running number) P1
บล็อกลงนามมาตรฐาน (เว้นบรรทัดลายเซ็น / ชื่อ-นามสกุล / ตำแหน่ง) แทรกคลิกเดียว + token เลขที่หนังสือแบบรันอัตโนมัติ (เช่น ศธ ๐๔xxx/ปี) ต่อยอดจากระบบ page token ที่มีอยู่
คุณค่า: สูงพยายาม: ★★ ✓ staticต่อยอด: pageTokens.ts, placeholderField
C3 รายชื่อผู้รับ / "สำเนาเรียน" ส่งทีละชุด (Distribution list) P2
จากรายชื่อผู้รับหลายหน่วยงาน สร้างหนังสือทีละฉบับ (เปลี่ยนคำขึ้นต้น/เรียน อัตโนมัติ) แล้วรวมเป็น ZIP → ต่อยอดโดยตรงจาก exportMailMerge.ts ที่ทำเสร็จแล้ว
คุณค่า: กลาง-สูงพยายาม: ★★ ✓ staticต่อยอด: exportMailMerge.ts
C4 เลขหน้า "หน้า ๑/๕" ไทย + บาทถ้วนในตารางอัตโนมัติ (Thai page no. + auto baht) P2
เลขหน้าแบบเลขไทย "หน้า ๑/๕" ในหัว/ท้ายกระดาษ + เปิด UI ให้ใช้ฟิลเตอร์ {{ยอดเงิน|baht}} (แปลงเป็น "หนึ่งพันบาทถ้วน") ที่มีในโค้ดแล้วได้สะดวกขึ้น
คุณค่า: กลางพยายาม: ★ ✓ static (ตรรกะมีแล้ว)ต่อยอด: thai/*, mergeFields.ts

→ ขั้นต่อไป
