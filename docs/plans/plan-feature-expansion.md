# wordhtml Feature Expansion Plan

## Objective
วิเคราะห์และวางแผนเพิ่ม feature ทั้งหมดให้ wordhtml เป็น "Word Processor ที่สมบูรณ์ใน Browser" โดยไม่ละทิ้ง concept privacy-first / client-side only

## Stage 1 — Competitive & Technical Research
- Research wordhtml.com ล่าสุดว่ามี feature อะไรบ้าง
- Research MS Word Online / Google Docs core features ที่ user คาดหวัง
- Research Notion/Linear UX patterns ที่น่าเอามาใช้
- วิเคราะห์ technical feasibility ใน browser (Web APIs, libraries)

## Stage 2 — Feature Analysis & Prioritization
- วิเคราะห์แต่ละ feature: impact, effort, technical complexity, dependencies
- จัดลำดับ priority ตาม user value + implementation cost
- ระบุ feature ที่ขึ้นกับ Stage 2 (architecture refactor) เสร็จก่อน

## Stage 3 — Detailed Roadmap
- สร้าง roadmap เป็น phases
- ระบุไฟล์ที่ต้องแก้/สร้างใหม่
- ระบุ Tiptap extensions ที่ต้องสร้าง
- ระบุ dependencies ใหม่

## Stage 4 — Deliverable
- รายงาน .md ที่ครบถ้วน
- อาจแปลงเป็น .docx ถ้าต้องการ
