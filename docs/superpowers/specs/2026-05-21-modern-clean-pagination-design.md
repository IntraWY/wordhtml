# Modern Clean Pagination Design

## Overview

Replace the current dashed-line page-break overlay (`PageBreakIndicator`) with a true Microsoft Word / Google Docs-style multi-page canvas display. Each page renders as a distinct white sheet with subtle shadow, centered on a gray canvas, with automatic content flow across pages driven by the existing pagination engine.

## Goals

- Eliminate dashed-line page break indicators entirely
- Display content as actual A4/Letter pages stacked vertically
- Maintain all existing editor functionality (rulers, drag-drop, keyboard, export, print)
- Leverage existing Phase 1 infrastructure (`pageNode`, `pageBody`, pagination engine, `usePagination`)

## Non-Goals

- Header/footer rich-text editing (Phase 2 — out of scope)
- Table/image splitting across pages (Phase 3 — out of scope)
- Different first page or odd-even headers (Phase 3 — out of scope)

## Visual Design

### Page Canvas

| Property | Value |
|----------|-------|
| Background (light) | `#f2f2f2` |
| Background (dark) | `#1a1a1a` or `var(--color-muted)` |
| Layout | Flex column, centered horizontally |
| Vertical padding | `24px` (`py-6`) |
| Horizontal alignment | `align-items: center` |

### Page Node (`.page-node`)

| Property | Value |
|----------|-------|
| Background | `#ffffff` (always white, even in dark mode) |
| Border radius | `2px` |
| Shadow (light) | `0 1px 3px rgba(0,0,0,0.10), 0 1px 2px rgba(0,0,0,0.06)` |
| Shadow (dark) | `0 4px 12px rgba(0,0,0,0.35)` |
| Margin bottom | `20px` (gap between pages) |
| Horizontal margin | `auto` (center within canvas) |
| Width | Exact A4/Letter px from `mmToPx` |
| Min-height | Exact A4/Letter px from `mmToPx` |

### Page Body (`.page-body`)

| Property | Value |
|----------|-------|
| Padding | Read from CSS custom properties set by parent `.page-node` |
| Overflow | `hidden` |
| Height | `100%` |

### Page Number

- Rendered via CSS `::after` pseudo-element on `.page-node[data-page-number]`
- Position: absolute, bottom `8px`, centered horizontally
- Color: `var(--color-muted-foreground)` (e.g., `#bbb` in light mode)
- Font size: `12px`
- `pointer-events: none; user-select: none`
- Hidden in print via existing `@media print` rules

## Component Architecture

### DOM Structure

```
EditorShell
└── PageCanvas (scrollable container)
    └── EditorContent (Tiptap ProseMirror)
        └── page-node[data-page-number="1"]
            ├── page-header (placeholder, contenteditable="false")
            ├── page-body (content container)
            └── page-footer (placeholder, contenteditable="false")
        └── page-node[data-page-number="2"]
            ├── page-header
            ├── page-body
            └── page-footer
```

### Key Components

**`PageCanvas`** (`src/components/editor/PageCanvas.tsx`)
- Serves as the scrollable outer container
- Applies gray canvas background and vertical padding
- Replaces the current `<article class="paper">` wrapper in `EditorShell`

**`PageWrapper`** (`src/components/editor/PageWrapper.tsx`)
- Kept for potential reuse (e.g., print preview, template rendering) but **not used** in the main editor path
- Tiptap's `pageNode.ts` renderHTML generates the equivalent DOM natively

**Tiptap `pageNode.ts`**
- Already renders `<div class="page-node">` with inline width/height styles
- Must be updated to set CSS custom properties for margins instead of applying padding directly to `.page-node`
- Keep `height:` (not `min-height:`) so `.page-body { height: 100% }` continues to resolve correctly
- Current inline padding on `.page-node` causes `.page-body { height: 100% }` to overflow the padded parent

**Tiptap `pageBody.ts`**
- Renders `<div class="page-body" data-page-body="true">`
- No changes needed to the extension itself

### CSS Custom Properties for Margins

Update `pageNode.ts` `renderHTML` to emit CSS variables:

```css
/* Inline style generated by pageNode.ts */
.page-node {
  --page-margin-top: 94px;
  --page-margin-right: 72px;
  --page-margin-bottom: 94px;
  --page-margin-left: 72px;
}
```

Update `.page-body` CSS to consume them:

```css
.page-body {
  padding-top: var(--page-margin-top, 0);
  padding-right: var(--page-margin-right, 0);
  padding-bottom: var(--page-margin-bottom, 0);
  padding-left: var(--page-margin-left, 0);
  overflow: hidden;
  height: 100%;
}
```

This fixes the height-overflow inconsistency between `pageNode.ts` and `PageWrapper.tsx`.

## Integration & Edge Cases

### EditorShell Changes

1. **Remove `PageBreakIndicator`** — no longer needed
2. **Replace `article.paper` with `PageCanvas`** as the editor content wrapper
3. **Move `articleRef`** from the old `<article>` to `PageCanvas` so `useEditorResize` still measures content height
4. **Move drag-and-drop listeners** from `article.paper` to `PageCanvas` or `window` so drops in the gap between pages still work

### Rulers

- **Horizontal ruler:** Stays at the top of the editing area, spanning the page width. Margin handles continue to adjust `pageSetup` globally.
- **Vertical ruler:** Should add tick marks or subtle indicators at page boundaries to show where each page begins, since the total content height now includes gaps.

### Empty State

`VisualEditor.tsx` already wraps empty documents in a `page-node` via `wrapInPageNode()`. No changes needed.

### Scroll & Navigation

- `usePagination` hook's `goToPage()` scrolls to the target page
- Must scroll to the **top** of the target `.page-node`, not the middle
- Add a small offset so the page doesn't sit flush against the viewport edge

### Dark Mode

- Canvas background switches to dark (`#1a1a1a`)
- Pages remain white (`#ffffff`) for realism
- Page shadow increases opacity in dark mode for visibility

### Print / Export

- Print stylesheet already has `.page-node` rules — add `margin: 0 auto` so pages center on the printed sheet
- All 5 export paths (HTML/ZIP/DOCX/PDF/MD) use `stripPaginationWrappers.ts` — no impact

### Keyboard & Cursor

- Arrow keys crossing page boundaries: ProseMirror handles this natively through `contenteditable`
- `Ctrl+Home` / `Ctrl+End`: Should jump to document start/end, not just current page
- Backspace at the top of page N: Content should flow back into page N-1; pagination engine handles this via `mergePage`

### Virtual Scroll

Existing `useVirtualScroll` with `content-visibility: auto` applies to `.page-node` elements naturally. May improve performance for documents with many pages.
